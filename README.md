# Образовательная игра-монополия для студентов

Веб-игра по типу «Монополии» для университетов. Преподаватель создаёт изолированную
комнату, студенты заходят по ссылке с телефона и ходят по очереди по общему полю из
24 клеток, отвечая на вопросы по предмету. Вся игровая логика — на авторитетном сервере.

Полное описание правил и архитектуры — в [PLAN.md](PLAN.md).

## Стек

- **Backend:** Python 3.12, FastAPI, SQLAlchemy 2.0 (async), Alembic, WebSocket.
- **Frontend:** React + TypeScript + Vite, Tailwind CSS, Zustand, нативный WebSocket.
- **БД:** PostgreSQL (локально допустим SQLite).
- **Экспорт:** CSV + Excel (openpyxl).

## Быстрый старт (локальная разработка, Windows)

Нужны: Python 3.12+, Node 20+, PostgreSQL (или SQLite по умолчанию).

### 1. Backend

```powershell
cd backend
py -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

# Настройте подключение к БД:
copy .env.example .env
# отредактируйте .env — DATABASE_URL (PostgreSQL или оставьте SQLite по умолчанию)

# Применить миграции (для PostgreSQL):
alembic upgrade head
# Для SQLite таблицы создаются автоматически при запуске.

uvicorn app.main:app --reload
```

Backend поднимется на `http://localhost:8000` (Swagger — `/docs`).

### 2. Frontend

```powershell
cd frontend
npm install
npm run dev
```

Откроется `http://localhost:5173`. Запросы `/api` и `/ws` проксируются на backend.

### 3. Игровой цикл

1. Преподаватель: `/teacher/login` → регистрация → создать **банк вопросов** →
   добавить вопросы вручную или импортировать `backend/sample_questions.csv`.
2. Создать **комнату** → скопировать ссылку вида `/join/КОД`.
3. Студенты открывают ссылку на телефоне, вводят имя/фамилию/группу.
4. Преподаватель нажимает **«Начать игру»**. Студенты ходят по очереди.
5. Экзаменационные ответы преподаватель проверяет вручную в очереди.
6. По завершении — **отчёт** с экспортом в CSV/Excel.

## Запуск через Docker

```bash
docker compose up --build
```

Поднимет PostgreSQL, backend (с авто-миграциями) и frontend (nginx).
Frontend — `http://localhost:5173`, backend — `http://localhost:8000`.

## Тесты

```powershell
cd backend
.\.venv\Scripts\python.exe -m pytest -q
```

Интеграционный тест поднимает приложение, регистрирует преподавателя, создаёт банк,
комнату, подключает двух студентов по WebSocket, запускает игру и проверяет отчёт.

## Структура

- `backend/app/game/` — авторитетный движок: конечный автомат ходов, резолверы клеток,
  таймеры, реестр комнат в памяти.
- `backend/app/ws/` — WebSocket-протокол и менеджер подключений (broadcast по комнатам).
- `backend/app/api/` — REST: auth, банки, вопросы, комнаты, отчёты.
- `backend/app/ai/` — интерфейс провайдера генерации вопросов (Фаза 8, post-MVP).
- `frontend/src/pages/student/` — мобильный экран студента (вход, лобби, игра, итоги).
- `frontend/src/pages/teacher/` — дашборд: банки, создание комнаты, управление, отчёт.
